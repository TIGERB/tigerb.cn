2. **新版小米网大秒系统**
  - 时间：2018.12 -- 2019.6
  - 项目简述：小米公司新品发售时，一般都会选择小米网作为新品的首次发售渠道。新品发售一般都是典型的高UV、高并发场景，所以如果把新品作为普通商品销售，会面临两个严峻的问题：1. 商城服务可能随时会被打垮 2. 商品可能会大量超卖。为了解决这个问题出现了大秒系统，大秒系统本质上是一个和业务相关的限流服务，大秒系统会给每位排队成功的用户下发令牌，成功获取令牌的用户才可以走购买流程。
    旧版大秒系统暴露除了如下的问题：
      1. 质量问题：老版本大秒共计由8个服务构成，服务太多、复杂度高、风险极大。且部分服务存在单点问题。
      2. 用户体验问题：用户排队成功获取令牌，加入购物车失败后无法重试。
      3. 效率问题：技术栈众多，研发维护成本高，运维维护成本高。语言涉及PHP、Go、Python，基础设施涉及MongoDB、Zookeeper、Redis、MySQL。
      4. 效率问题：运营后台多，配置流程复杂，且不支持全球化。
      5. 架构问题：监控体系薄弱纯自研，单点切维护成本高。
  - 项目目标：
      1. 缩减服务数：由8个服务缩减到1～2个服务。
      2. 用户体验：成功获取令牌的用户，下单异常支持重试。
      3. 技术栈统一：统一技术栈为Go、Redis、Mysql，由7个技术栈缩减到3个核心技术栈。
      4. 效率提升：收敛3个运营后台，并支持国际化。
      5. 架构问题：可观测全面接入Promethues体系。

  - 个人职责:项目负责人
    1. 负责服务整体架构设计。
    2. 负责核心代码建模。
    3. 负责该服务人才梯队建设，并指导对应同学完成项目落地。
    4. 负责项目关键节点把控。
  
  - 项目收益：
    1. 服务质量提升：从每月必出故障到季度0故障。
    2. 技术栈统一：技术栈完成去PHP、Python、MongoDB、zk依赖。
    3. 效率提升：研发由日常维护8个服务缩减到到1个服务；运营实现1个运营后台完成活动配置。
    4. 其他收益：IDC成本降低55%。
